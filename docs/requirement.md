## Description of the App
For this project, the team would like to create an internal social networking app for employees in a growing startup company. The name of the app is TeamConnect. The goal of the app is to help employees communicate, build relationships, stay informed, and feel more connected to the company.
This idea is practical because employee engagement matters to business performance. Gallup reported that engaged employees drive stronger business outcomes, and recent Gallup reporting also shows that U.S. employee engagement has remained low, which means many organizations still have a real need to improve connection and morale at work.  
### Required Core Features
The app will include the core features described in the project sample. These include a signup process, login form, logout feature, session control, user profiles with uploaded thumbnails, a member directory, adding members as friends, public and private messaging, and styling of the project.
The signup feature will allow employees to create their accounts. The login and logout features will let users securely enter and leave the system. Session control will help manage access after login. User profiles will let employees add a profile photo and short biography so coworkers can know each other better. The member directory will help users search or browse colleagues across the company. The friend feature will allow employees to build internal connections. Public and private messaging will support both open interaction and direct person-to-person communication. These features follow the required structure of the sample social networking application.
### Additional Features for Internal Company Use
Because this is an internal company platform, TeamConnect will also include several extra features that make it more useful in a real workplace.
The first feature is an announcement section for company administrators. This area can be used to post official internal updates such as holidays, policy changes, deadlines, or event reminders. A central place for announcements can help employees stay informed and reduce communication gaps.
The second feature is a company news section. This section can share business updates, employee highlights, project milestones, and company events. A tool like this can strengthen internal community because enterprise social platforms help employees connect with leaders and coworkers, support two-way dialogue, and build stronger organizational culture.  
The third feature is an appraisal and recognition tool. Employees and managers can use this feature to recognize achievements, celebrate good work, and support appreciation across teams. Gallup states that recognition helps employees feel valued and can increase engagement, productivity, loyalty, and retention. Gallup also reported that employees who receive high-quality recognition are less likely to leave their jobs.  
The fourth feature is a knowledge base. This part of the app will contain onboarding materials, company policies, internal guides, process instructions, and FAQs. A knowledge base is valuable because documentation encourages knowledge sharing, helps team members understand how work is done, and supports consistency in repeated tasks. Atlassian also notes that knowledge sharing helps organizations become more agile and adaptable.  
The fifth feature is a complaint, feedback, and suggestion tool. This section will let employees raise concerns, share ideas, and provide suggestions for improvement. Frequent feedback and recognition together can improve engagement more than feedback alone. Gallup reported that employees who receive both frequent feedback and frequent recognition are significantly more likely to be engaged.  
## Technical Requirements
This project will focus on the user interface only. The application will be built as a React front-end prototype and will not require a live backend, database, or production authentication service for the class demo.

### Front-End Scope
The UI should demonstrate the full user journey for the main screens and interactions. Pages such as signup, login, profile, directory, messaging, announcements, news, recognition, knowledge base, and feedback should behave like a working product, but their data can be simulated in the browser.

### Technology Direction
React will be used to build reusable components and page-level views. Styling can be handled with standard CSS, CSS modules, or another lightweight front-end approach chosen by the team. Navigation, filtering, form entry, and modal behavior should be handled on the client side.

### Data Strategy for Demo
All application content should come from mock data stored in local files. This includes employee profiles, friend connections, message threads, announcements, company news, recognition posts, knowledge base articles, and feedback items. Form actions such as login, posting, searching, and sending a message can update local React state without saving to a server.

### Demo Constraints
Authentication should be presented as a simulated UI flow only. File uploads, profile photos, and message history can use placeholders or mock previews. The goal is to show a polished and believable front-end experience rather than complete backend integration.

### Why This App Is a Good Fit
TeamConnect is a strong fit for this project because it combines social networking features with practical internal business tools. Instead of using separate systems for communication, recognition, knowledge sharing, and employee voice, the startup could use one internal platform.
From a technical view, React is a strong fit for this project because it supports component reuse, state-driven interfaces, and fast UI prototyping. A front-end-only implementation also reduces complexity for the current milestone while still allowing the team to demonstrate the complete product concept with realistic screens and interactions.

## Mock Data Requirement
For demo purposes, the project should include a reusable mock dataset that covers the main features of TeamConnect. At minimum, the demo data should include:

- employee accounts and profiles
- friend and team connections
- direct and public message samples
- announcements and company news posts
- recognition entries
- knowledge base articles
- complaints, feedback, and suggestions

This mock data should be stored in a central location so React components can import and render it consistently across pages.

## Delivery Requirements
The completed app should be pushed to GitHub so the repository can be reviewed and shared. The team should also record a short screencast between 3 and 5 minutes long that walks through the major screens, explains the UI flow, and demonstrates the mock data in use. Submit the video link with the project. Only one team member needs to submit the app video.

## Summary of the App Idea
In summary, TeamConnect is an internal employee social networking app for a growing startup. It includes all the main social networking features required in the project sample, such as signup, login, logout, profiles, directory, friend connections, messaging, session control, and styling. It also adds workplace-focused functions such as announcements, company news, appraisal and recognition, a knowledge base, and a complaint/feedback/suggestion tool.
This version of the project will focus on a polished React user interface supported by local mock data rather than backend development. Even as a front-end prototype, the app can still show how employees would communicate, stay updated, recognize each other’s work, find useful internal information, and share feedback with the company.
